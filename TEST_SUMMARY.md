## 认证流程

1. **启动流程**：
   - AppWrapper检查本地认证状态
   - 有效认证 → 进入主页
   - 无效认证 → 显示登录界面

2. **登录流程**：
   - 用户输入凭据
   - API尝试登录（可降级到离线模式）
   - 认证信息保存到Hive
   - 小部件数据更新
   - 登录成功回调触发
   - 跳转到主页

3. **注销流程**：
   - SettingsScreen中点击注销
   - 确认对话框弹出
   - Hive认证数据清理
   - 小部件数据更新
   - 返回登录界面

## 技术架构

### 状态管理
- Provider模式，包含4个ChangeNotifierProvider：
  1. `ScheduleProvider` - 课程状态
  2. `TodoProvider` - 待办事项
  3. `ThemeProvider` - 主题配置
  4. `CourseProvider` - 课程数据

### 数据存储
- Hive本地数据库，包含5个数据盒子：
  1. `reminders` - 提醒事项
  2. `todos` - 待办事项
  3. `courses` - 课程信息
  4. `settings` - 应用设置
  5. `api_auth` - 认证信息

### 动画系统
- 页面切换：PageView + 滑动手势
- 卡片入场：交错动画（平移+透明度+缩放）
- 活动指示：脉冲效果（透明度循环）
- 状态显示：动态渐变阴影
